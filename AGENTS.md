# AGENTS.md

이 문서는 이 저장소에서 작업하는 에이전트와 기여자가 따라야 할 기본 규칙입니다.

## 프로젝트 개요

- 프로젝트명: CaregiverCall
- 목표: ESP32-C3 Super Mini 보드와 Matter를 이용해, 집 안에 있는 지체장애인이 버튼을 눌러 외부 보호자를 호출할 수 있는 보조기기를 만듭니다.
- 주요 시험 환경: Google Nest Mini를 Matter Hub로 사용합니다.
- 현재 저장소는 초기 상태입니다. 펌웨어 스택, Matter 연동 방식, 빌드 도구, 테스트 도구가 추가되면 이 문서를 함께 갱신하세요.

## 제품 원칙

- 이 기기는 보조기기입니다. 기능 구현보다 사용자의 안전, 호출 신뢰성, 접근성을 우선합니다.
- 버튼 입력은 손 떨림, 긴 누름, 중복 입력, 실수 입력 가능성을 고려해 설계합니다.
- 호출 실패, 네트워크 끊김, 전원 재시작 같은 예외 상황을 명시적으로 다룹니다.
- 사용자에게 필요한 피드백(예: LED, 부저, 진동 등)은 단순하고 일관되게 설계합니다.
- 보호자가 스마트폰에서 호출을 확인하면, 사용자가 확인 사실을 알 수 있도록 Nest Mini 음성 안내와 가능한 로컬 피드백을 제공합니다.
- 개인정보, 위치, 보호자 연락 정보 같은 민감한 데이터는 최소한으로 다루고 안전하게 보관합니다.

## Matter 연동 원칙

- Google Nest Mini를 기본 시험용 Matter Hub로 사용합니다.
- Google Nest Mini는 보호자 확인 사실을 사용자에게 알려주는 음성 피드백 장치로도 사용합니다.
- 구현은 Google 전용 API나 특정 허브의 비표준 동작에 의존하지 않습니다.
- 가능한 한 Matter 표준 클러스터, 표준 속성, 표준 커미셔닝 흐름을 우선합니다.
- 다른 Matter Hub(예: Apple Home, Samsung SmartThings, Home Assistant 등)에서도 동작할 수 있도록 허브 종속 가정은 문서화합니다.
- 허브별 차이가 필요한 경우 공통 Matter 동작을 기본값으로 두고, 허브별 처리는 분리된 호환성 계층으로 제한합니다.
- Google Home 자동화나 Broadcast 기능은 편의 기능으로 취급하고, 안전 필수 동작의 유일한 보장 수단으로 의존하지 않습니다.

## 작업 원칙

- 기존 코드와 문서의 스타일을 먼저 확인하고, 새 파일도 그 흐름에 맞춥니다.
- 구현 세부사항이 열려 있으면 단순하고 보수적인 선택을 우선합니다.
- 불필요한 리팩터링, 포맷 변경, 대규모 구조 변경은 피합니다.
- 사용자나 다른 작업자가 만든 변경사항을 되돌리지 않습니다.
- 동작 변경이 있으면 가능한 한 작고 검증 가능한 단위로 나눕니다.

## 파일 및 문서 규칙

- 새 문서는 기본적으로 Markdown을 사용합니다.
- `src/`는 펌웨어 소스 코드, `lib/`는 프로젝트 내부 라이브러리 또는 외부 라이브러리 래퍼, `docs/`는 제품 요구사항과 매뉴얼을 담습니다.
- 저장소에 별도 규칙이 생기기 전까지 파일명과 디렉터리명은 명확한 영어 이름을 사용합니다.
- 프로젝트 구조, 실행 방법, 테스트 방법이 정해지면 README.md와 이 파일을 함께 갱신합니다.
- 제품 요구사항이나 사용자 흐름이 바뀌면 `docs/product_requirements.md`를 함께 갱신합니다.
- 환경설정, 컴파일, 업로드 절차가 바뀌면 `docs/development_manual.md`를 갱신합니다.
- Google Home 연결 절차가 바뀌면 `docs/setup_manual.md`를 갱신합니다.
- 실제 사용자 흐름이나 보호자 사용 방법이 바뀌면 `docs/user_manual.md`를 갱신합니다.
- 중요한 작업이 완료되면 실제 인수인계 문서를 `docs/session_handoffs/` 아래에 새로 만들고, `SESSION_HANDOFF.md`에는 색인 링크와 짧은 요약만 추가합니다.
- 임시 파일, 빌드 산출물, 로컬 설정 파일은 커밋하지 않습니다.

## Python 환경 원칙

- Python을 사용하는 도구나 스크립트는 저장소 로컬 `.venv` 가상환경에서 실행합니다.
- 시스템 Python에 패키지를 직접 설치하지 않습니다.
- Python 의존성은 `requirements.txt`에 기록합니다.
- 저장소를 처음 클론한 사용자는 환경 설정 스크립트로 `.venv`를 만들고 의존성을 설치할 수 있어야 합니다.
- Windows에서는 `.\scripts\setup_python_env.ps1`, macOS/Linux에서는 `./scripts/setup_python_env.sh`를 사용합니다.
- `.venv`와 Python 캐시 디렉터리는 커밋하지 않습니다.

## 개발 및 검증

- 빌드, 린트, 테스트 명령이 추가되면 아래에 기록합니다.
- 명령이 아직 없을 때는 실행했다고 가정하지 말고, 검증하지 못한 내용을 작업 결과에 명확히 적습니다.
- 펌웨어 변경은 가능한 경우 실제 보드 또는 명시된 타깃 환경에서 검증합니다.
- Matter 연동 변경은 페어링, 재연결, 전원 재시작 후 동작을 확인합니다.
- Google Nest Mini에서 먼저 시험하되, 변경이 Matter 표준 동작을 벗어나지 않는지 함께 확인합니다.
- 호출 관련 로직은 버튼 입력부터 보호자 알림까지의 전체 흐름을 기준으로 검증합니다.
