# Setup Manual

이 문서는 CaregiverCall 기기를 Google Home에 연결하고 Google Nest Mini를 이용해 시험하는 절차를 설명합니다.

## 준비물

- CaregiverCall 기기
- Google Nest Mini
- Google Home 앱이 설치된 스마트폰
- 보호자 스마트폰
- 2.4 GHz Wi-Fi 네트워크
- Matter 커미셔닝 QR 코드 또는 페어링 코드

## 사전 조건

- Google Nest Mini가 Google Home 앱에 등록되어 있어야 합니다.
- 스마트폰과 Nest Mini가 같은 집(Home) 구성에 있어야 합니다.
- CaregiverCall 펌웨어가 Matter 커미셔닝 모드를 지원해야 합니다.
- 보호자 알림과 확인 흐름은 Google Home 자동화 또는 별도 연동 방식으로 구성해야 합니다.

## Google Home에 기기 연결하기

1. CaregiverCall 기기에 전원을 연결합니다.
2. 기기가 커미셔닝 모드인지 확인합니다.
3. Google Home 앱을 엽니다.
4. 기기 추가 메뉴에서 Matter 기기 추가를 선택합니다.
5. 기기의 QR 코드 또는 페어링 코드를 입력합니다.
6. Google Home 앱 안내에 따라 방과 이름을 지정합니다.
7. 페어링 완료 후 기기의 호출 상태가 Google Home에서 인식되는지 확인합니다.

## 보호자 알림 구성

초기 시험에서는 가급적 Google Home 자동화만 사용합니다.

1. 호출 상태가 변경될 때 보호자 스마트폰에 알림을 보내도록 자동화를 구성합니다.
2. 알림에는 사용자가 호출했다는 사실과 확인 동작이 명확히 보이도록 합니다.
3. 보호자가 확인 버튼을 누르면 확인 상태가 기록되거나 집 안 시스템으로 전달되도록 구성합니다.
4. 확인 상태를 트리거로 Nest Mini가 사용자에게 음성 안내를 하도록 구성합니다.

예시 음성 안내:

```text
보호자가 호출을 확인했습니다.
```

## Nest Mini 음성 안내 확인

1. CaregiverCall 버튼을 누릅니다.
2. 보호자 스마트폰에서 호출 알림이 도착하는지 확인합니다.
3. 보호자 확인 버튼을 누릅니다.
4. Nest Mini가 확인 안내를 말하는지 확인합니다.
5. 기기의 LED, 부저, 진동 등 로컬 피드백이 있다면 함께 확인합니다.

## 주의사항

- Google Home 자동화와 Nest Mini 음성 안내는 편의 기능입니다. 안전 필수 동작의 유일한 보장 수단으로 의존하지 않습니다.
- Google Home 자동화만으로 보호자 확인 버튼 흐름을 완전히 구현할 수 있는지는 실제 기기 등록 후 검증해야 합니다.
- Nest Mini가 오프라인, 음소거, 방해 금지, Downtime, 네트워크 장애 상태이면 음성 안내가 실패할 수 있습니다.
- 다른 Matter Hub에서도 기본 호출 상태가 해석될 수 있도록 Google 전용 설정은 문서화해서 분리합니다.

## 문제 해결

| 문제 | 확인할 것 |
| --- | --- |
| Matter 기기가 검색되지 않음 | 기기가 커미셔닝 모드인지, 전원이 안정적인지, QR/페어링 코드가 맞는지 확인 |
| Google Home에 추가되지만 자동화가 동작하지 않음 | 자동화 트리거가 기기의 실제 상태 변경과 일치하는지 확인 |
| 보호자 알림이 오지 않음 | 스마트폰 알림 권한, Google Home 집 구성원, 네트워크 상태 확인 |
| Nest Mini가 말하지 않음 | Nest Mini 전원, 볼륨, 방해 금지, Downtime, 네트워크 상태 확인 |
| 호출이 반복됨 | 버튼 디바운스, 긴 누름 처리, 호출 재시도 정책 확인 |
