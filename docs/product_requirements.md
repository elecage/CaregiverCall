# Product Requirements

## 목적

CaregiverCall은 집 안에 있는 지체장애인이 물리 버튼을 눌러 외부 보호자를 호출하고, 보호자가 확인했음을 사용자에게 다시 알려주는 보조기기입니다.

## 기본 호출 흐름

1. 사용자가 ESP32-C3 Super Mini 기반 호출 버튼을 누릅니다.
2. 기기는 Matter를 통해 집 안의 Matter Hub에 호출 상태를 전달합니다.
3. 기본 시험 환경에서는 Google Nest Mini와 Google Home을 사용해 보호자의 스마트폰으로 호출 알림을 보냅니다.
4. 보호자는 스마트폰에서 확인 버튼을 누릅니다.
5. 보호자 확인 상태가 집 안 시스템으로 전달됩니다.
6. Google Nest Mini가 사용자에게 보호자가 확인했음을 음성으로 알려줍니다.
7. 기기는 LED, 부저, 진동 등 로컬 피드백이 있다면 호출 완료 또는 확인 완료 상태를 함께 표시합니다.

## 보호자 확인 요구사항

- 보호자 알림에는 사용자가 호출했다는 사실과 확인 동작이 명확히 보여야 합니다.
- 보호자가 확인 버튼을 누르면 확인 시각을 기록할 수 있어야 합니다.
- 보호자 확인 후 사용자에게 전달되는 안내는 짧고 명확해야 합니다.
- 예시 안내: "보호자가 호출을 확인했습니다."
- 확인 응답이 일정 시간 안에 오지 않으면 재알림, 실패 표시, 대체 연락 수단을 고려합니다.
- 보호자 확인 피드백은 음성 안내만으로 끝내지 않고, 가능한 경우 로컬 표시와 함께 제공합니다.

## Google Home / Nest Mini 연동 요구사항

- Google Nest Mini는 기본 시험용 Matter Hub이자 사용자 음성 피드백 장치로 사용합니다.
- Google Home 자동화의 스마트폰 알림 기능과 Google Assistant Broadcast 기능을 후보 경로로 검토합니다.
- Google Home 자동화는 편의 기능으로 취급하고, 호출 실패 시 위해가 생길 수 있는 유일한 안전 장치로 의존하지 않습니다.
- Nest Mini가 방해 금지, Downtime, 네트워크 장애, 계정 설정, Google 서비스 상태 등의 이유로 음성 안내를 하지 못할 수 있음을 고려합니다.
- Google Home에 종속되는 자동화 정의는 별도 문서나 설정 예제로 분리합니다.

## Matter 호환성 요구사항

- 구현은 특정 Google 전용 API에 묶이지 않고 Matter 표준 동작을 우선합니다.
- 1차 개발 목표는 Google Home에서 Matter On/Off Switch로 표시되는 것입니다.
- Google Nest Mini에서 먼저 시험하되, 다른 Matter Hub에서도 호출 상태가 해석될 수 있도록 모델링합니다.
- 허브별 자동화나 알림 차이는 호환성 계층 또는 문서화된 운영 설정으로 분리합니다.
- Matter 디바이스 타입과 클러스터 선택은 호출 상태, 확인 상태, 장애 상태를 가장 명확하게 표현할 수 있는 표준 조합을 우선합니다.

## 실패 및 예외 흐름

- 버튼 입력이 감지되었지만 Matter Hub로 전달하지 못한 경우 사용자에게 로컬 실패 피드백을 제공합니다.
- 보호자 알림 전송 여부를 확인할 수 없는 경우 재시도 또는 미확인 상태를 유지합니다.
- 보호자가 확인하지 않은 상태와 확인 완료 상태를 명확히 구분합니다.
- 전원 재시작 후에는 이전 호출 상태를 복원하지 않고 대기 상태로 초기화합니다.
- 네트워크 복구 후 중복 호출이나 오래된 확인 알림이 발생하지 않도록 타임스탬프 또는 호출 ID를 사용합니다.

## 검증 시나리오

- 버튼 1회 누름으로 보호자 알림이 생성되는지 확인합니다.
- 손 떨림이나 연속 입력으로 중복 호출이 과도하게 발생하지 않는지 확인합니다.
- 보호자 확인 후 Nest Mini가 확인 안내를 말하는지 확인합니다.
- Nest Mini가 오프라인이거나 방해 금지 상태일 때 로컬 피드백이 적절한지 확인합니다.
- Google Nest Mini 외의 Matter Hub에서 기본 호출 상태가 인식 가능한지 확인합니다.
- 전원 재시작, Wi-Fi 끊김, Matter 재연결 후 상태가 일관적인지 확인합니다.
